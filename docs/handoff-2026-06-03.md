## What changed since 2026-06-02

This session ran in 4 phases. Bigger arc than the prior session: ORE-10908
went from 27× Konclude to 4.32×.

### Shipped (durable code changes)

* **Phase 7 — per-class label heuristic** (commits `100a52c` → `46aa76d`,
  10 commits). HermiT-style sound non-subsumption pruner: per-class
  wedge satisfiability builds a label cache once; orchestrator skips
  `subsumes_via_tableau` when `D ∉ labels(C)` (sound counterexample-model),
  verifies when `D ∈ labels(C)`. New: `HyperEngine::satisfiability_labels`,
  `LabelOracle`, `HyperCache::classify_labels`, `PreparedOntology::classify_labels`,
  three new `ClassificationStats` counters, env gate `RUSTDL_LABEL_HEURISTIC`.
  Structural canary at `tests/label_heuristic_canary.rs`.
  - GALEN classify wall: 684 s → 455 s (−33 %).
  - ORE-10908: 27 s → 19 s (−29 %), Konclude ratio 17× → 12×.
  - Prune rate 89-100 % across all measured workloads.
  - See `docs/phase7-results.md`.

* **Phase 8 — decouple label-cache deadline** (commits `30b641c` +
  `420f78d`). The Phase 7 cache build was using `--pair-timeout-ms`
  (200 ms) for the per-class wedge call; 5 % of ORE-10908 classes
  stalled at that budget and each became NoVerdict → tier walk
  miss-bucket bloated by ~28 ms × ~38 pairs per staller. Fix: separate
  cache-build deadline (`RUSTDL_LABEL_CACHE_TIMEOUT_MS`, default 1000
  ms). All 1003 ORE-10908 misses convert to prunes.
  - ORE-10908: 19.32 s → **7.48 s (−61 %)**. Konclude ratio 12× → **4.32×**.
  - Per the user's session-start target: ≤ 5× Konclude — **HIT**.
  - Default tuned from initial 5000 ms to 1000 ms (5000 wasted
    ~5 s on ORE-15672 with no gain). Empirical sweet spot.
  - See `docs/phase8-results.md`.

* **Phase 10 — fresh full-corpus head-to-head** (commit `b8bf670`):
  `docs/perf-2026-06-03-konclude-vs-rustdl.md`. Supersedes the
  June-02 head-to-head. All 19 corpus ontologies measured.

### Investigated + characterized (no code shipped)

* **Phase 9 — ORE-15672 NoVerdict cluster recon** (commit `ded4368`,
  close-out `f5a055e`). Goal: identify what construct made ORE-15672's
  residual 17× gap, recommend saturator extension. Findings:
  - "46-class NoVerdict cluster" framing from Phase 8 was a miscount.
    Only **3 classes** actually NoVerdict; `misses=46` is tier-walk
    consultation events (each consulted ~15 times).
  - **No dominant construct**. A near-identical sibling (`e-usage-situation`)
    resolves to Sat. Discriminator is search-budget exhaustion on
    3 `∃proper-part.X` hops over a transitive+inverse role with
    inherited 5-conjunct body.
  - Nominal / cardinality / inverse-role extensions all structurally
    ruled out (sibling-class disambiguation).
  - **Recommendation: accept the gap.** Only structural lever is
    Konclude-style sub-model caching (dead-end §2) — uncertain benefit
    on one small ontology.
  - Dead-end §18 documents the construct-by-construct elimination.

## Soundness contract (unchanged from 2026-06-02, restated for context)

* Hypertableau wedge is the default accelerator. FP = 0 throughout.
* `trust_sat` default-on: wedge concludes "not subsumed" from its own
  Sat verdict without consulting the tableau. **Sound, near-complete-
  but-not-guaranteed-complete approximation.** Set
  `RUSTDL_HYPERTABLEAU_TRUST_SAT=0` for slower complete behaviour.
* `RUSTDL_HYPER_TRUST_SAT_MIN_MS` (Phase 1, default 0) — opt-in only.
* **New as of Phase 7**: `RUSTDL_LABEL_HEURISTIC=0` disables the
  per-class non-subsumption pruner (default ON). Sound by the
  model-as-counterexample argument: a Sat wedge verdict produces a
  completion graph that IS a witness model; D ∉ labels(C) in that
  model is a sound non-subsumption proof.
* **New as of Phase 8**: `RUSTDL_LABEL_CACHE_TIMEOUT_MS` (default 1000)
  caps per-class cache-build wall. Set to 0 for unbounded.
* Soundness gate: `tests/konclude_closure_diff.rs` on alehif +
  ORE-10908 + ORE-15672 + GALEN + notgalen. FP=0 in CI.

## Lessons from this session

The Phase 5 → 6 → 7 → 8 arc reinforces the meta-lesson from prior
sessions: **recon before structural change.**

1. **Phase 5 recon localized GALEN's regression to tier-walk**,
   prompting Phase 6's visited-set dedup (−9 % wall).
2. **Phase 5 T3b's counter set missed wedge_refuted_pairs**, so the
   wall budget on GALEN looked tableau-bound when actually wedge-bound.
   Phase 7's label heuristic shipped a −33 % GALEN win **the prior
   counter set couldn't see**. Surprise wins live in counter blindspots.
3. **Phase 8 recon localized ORE-10908's residual to deadline-bound
   stallers, not structural cost.** Same recon → surgical fix pattern.
   1000 ms vs 5000 ms tuning came from a measurement, not a guess.
4. **Phase 9 recon falsified the Phase 8 framing** ("46-class cluster"
   was 3 classes consulted 15 times). Even close-out work benefits
   from instrumentation.

Across 138 commits this period: **every shipped perf win started with
a recon doc; every avoided dead-end started with the same**. The
discipline scales.

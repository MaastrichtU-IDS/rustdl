## STAGE 2/3 ‚Äî pilot (stratified: all 148 OutOfFragment <5MB + 34 Horn + 50 PureEl = 232)

Each reasoner bounded at 300 s; oracle = Konclude‚à©HermiT agreement; disagreements excluded.

### Harness bugs found + fixed during self-test (pizza ground-truth) ‚Äî all were artifacts
1. **XML entity decode**: rustdl emits raw `&apos;` in IRIs; Konclude/HermiT decode. Fixed (html.unescape all IRIs).
2. **`equiv` lines**: rustdl prints equivalence groups as `equiv\tA\tB...` not as `direct` edges; harness dropped them ‚Üí spurious MISSED (incl. all of pizza's "4 misses", 10952's "842", 11623's "46"). Fixed (expand equiv cliques).
3. **HermiT non-UTF-8 output**: HermiT `-c` mangles accented IRIs (`√©`‚Üí`?`) ‚Üí spurious oracle disagreement + FP (1325: FP=325 vanished). Fixed (`-Dfile.encoding=UTF-8`).
4. **rustdl Tseitin synthetics + signature**: restrict closures to the oracle's named-class signature.
5. **FP definition**: C1 uses `FP_strict = rustdl ‚àñ (Konclude ‚à™ HermiT)` (assert-by-NEITHER), not vs intersection.

All FP/MISSED judged at BOTH levels: classify-output closure AND per-pair `rustdl subclass`
(the calculus). classify_gap = subclass=yes (rendering gap); calculus_fn = subclass=no (genuine FN).

### PIVOTAL ANSWERS

**(1) C1 ‚Äî any rustdl FP>0?**
- **Calculus-level (per-pair `rustdl subclass`): FP = 0** across all 187 diffed ontologies ‚Äî **modulo 3 unadjudicated pairs on ore_ont_6423** (`OBI_*‚äëNCBITaxon_2759`), whose `subclass` probe DNFs even unbounded at 300 s, so it can be neither confirmed artifact nor confirmed unsound. Every other FP-candidate pair checked is `subclass=no` (calculus_unsound=0). C1 holds for the reasoner modulo that one unresolved ontology.
- **classify-OUTPUT level: FP_strict>0 on 6 ontologies** (13723, 2232, 4897, 5940, 9941, 6423 = 30/30/35/4/62/3 pairs). For 5 of 6 (all but 6423, whose `subclass` and re-run both DNF) the pairs are `subclass=no` AND vanish under `RUSTDL_SNAPSHOT_CAPTURE=0` ‚Üí **the default classify config's snapshot cache emits spurious subsumptions** the calculus itself rejects, with no warning. This is a real, reproducible **default-config unsoundness at the output level** ‚Äî surfaced only by ORE; the tuned 26-ont corpus never exercised it. (Root: snapshot cache on classes defined via union/`EquivalentClasses(X, ‚äî)`.)

**(2) C2 ‚Äî silent false-negatives (MISSED>0 with timed_out=0)?**  YES, demonstrated.
- 21 ontologies MISSED>0; **16 SILENT** (timed_out=0, no incompleteness signal), 5 flagged.
- Mechanism split (per-pair subclass, capped 60 pairs/ont, 15s/pair):
  - **classify_gap (subclass=yes ‚Äî rustdl entails it, `classify` just doesn't print it): 259 pairs.** A `classify`-command rendering/orchestration gap (LHS-union & equivalence-group edges not emitted as `direct`). Affects pizza-style equivalences, `SubClassOf(‚äî(A,B),C)`, etc.
  - **calculus_fn (subclass=NO, oracle=yes, timed_out=0 ‚Äî GENUINE silent C2 false-negative): 72 pairs across 4 ontologies** ‚Äî the strongest result, IMPOSSIBLE to see on the MISSED=0 tuned corpus. **VALIDATED as calculus (not silent-channel artifact):** the exemplar of each cluster re-checked with `RUSTDL_HYPERTABLEAU_TRUST_SAT=0 RUSTDL_LABEL_HEURISTIC=0 RUSTDL_SNAPSHOT_CAPTURE=0` + full unbounded tableau still returns `no` ‚Äî so it is genuine calculus incompleteness, not a trust_sat/label/snapshot mask:
    - `ore_ont_9054` (photography): 60 pairs, e.g. `120RollFilm ‚äë MediumFormatFilm` ‚Äî driven by `‚àÉhasFormat.(Format ‚äì DataHasValue(height,60) ‚äì DataHasValue(width,60))`; rustdl's deliberate **datatype under-approximation (Phase D1: unrecognized data ranges silently dropped)** breaks the existential match ‚Üí silent miss.
    - `ore_ont_15273` + `ore_ont_6437` (SAO neuroscience, 2 distinct onts): 5 pairs each, `sao1785599611 ‚äë {Continuant, Entity, Object, ...}` ‚Äî rustdl fails `sao-943768255 ‚äë snap#Continuant` (cross-ontology BFO chain), inherited downstream.
    - `ore_ont_9941` (xenc): 2 pairs.
  - **query_err (subclass timed out at 15s ‚Äî UNRESOLVED): 183 pairs.** Concentrated in hard-SROIQ onts (778, 3250, 13737, 7517, 7339, 15682, 9786). So calculus_fn=72 is a **lower bound**; some unresolved pairs may also be calculus FNs.
- trust_sat / channel differentials (on the cheap cases): 12253's silent miss became a *flagged* timeout under `RUSTDL_HYPERTABLEAU_TRUST_SAT=0` (the trust_sat `Sat` channel was masking it) ‚Äî confirms the C2 "silent channel" architectural hole the calibration matrix predicted.

**(3) C3 ‚Äî comparative anytime (complete reasoner DNF, rustdl sound partial)?**  Partial / weak.
- **4 ontologies where HermiT hit the 300s timeout but rustdl AND Konclude completed** (16666/2792/3077/3917, ~3800‚Äì5600 classes): rustdl FP=0 vs Konclude, finished in 143‚Äì223 s. rustdl is more robust than HermiT here, BUT Konclude completes them in 1‚Äì2 s, so it is NOT "complete reasoners give nothing." ### Does ORE rescue a research-track claim?

**Mixed ‚Äî it does the two things the tuned corpus could not, both honestly negative-leaning for the current default but valuable as findings:**

1. **C2 non-vacuous, and it fires:** ORE exhibits **genuine silent calculus-level C2 false-negatives** (‚â•72 pairs, 4 onts; lower bound) that produce a wrong+unflagged hierarchy under the default config ‚Äî the headline C2 vulnerability the matrix could only hypothesize. Characterizable: the largest cluster is the **Phase D1 datatype under-approximation** (sound-by-design drop becomes a *silent* incompleteness on real datatype-defined ORE classes). This is a concrete, fixable calibration target ‚Äî research-track material IF framed as "we found and characterized the silent channel," not "we are calibrated."
2. **C1 nuance:** the *reasoner calculus* is sound (FP=0, 187 onts) ‚Äî the robust headline survives at scale. But the **default `classify` config is unsound at the output level** via the snapshot cache (6 onts, spurious edges, no warning), reproducibly fixed by `RUSTDL_SNAPSHOT_CAPTURE=0`. A reviewer re-running ORE would hit this; it must be fixed or disclosed before any FP=0 claim.
3. **C3 comparative anytime: NOT supported** even at ORE scale ‚Äî no ontology where all complete reasoners fail and rustdl wins; Konclude completes everything attempted. Confirms the resource-track standing on C3.

**Net:** ORE confirms the resource-track standing (C1 calculus-sound, C3 unsupported, EL/Horn fast) AND surfaces two concrete default-config defects (snapshot-cache output-unsoundness; datatype silent-FN) that are publishable as a characterization/fix ‚Äî but it does NOT hand a clean research-track win; it raises the bar by exposing real silent misses that must be addressed.
